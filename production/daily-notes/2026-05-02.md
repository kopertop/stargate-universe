# Stargate Universe: Black Screen on "New Game" Fix

## Problem
Clicking "New Game" resulted in a black screen. The opening-cinematic scene
mounted successfully (music played, 3D objects added to scene) but the
lifecycle `update()` callback was never called.

## Root Cause
The `GameLoop.start()` method was never invoked. In `src/game/app.ts`,
the `start()` function only called `loadScene()` but never started the
game loop:

```ts
const start = () => loadScene(options.initialSceneId);
```

Since `GameLoop` uses `requestAnimationFrame` internally, without calling
`start()` the `onUpdate`/`onRender` callbacks never fire, meaning scene
lifecycle `update()` methods are never called and nothing renders.

## Fix
Changed `start()` to call `loop.start()` before `loadScene()`:

```ts
const start = () => {
  loop.start();
  return loadScene(options.initialSceneId);
};
```

Key insight: `loop.start()` must be called **before** `loadScene()` so
the first loaded scene receives update ticks immediately. The previous
order (loadScene then start) would also work but creates a brief window
where the loaded scene has no update ticks.

## Debugging Approach
When a scene mounts but nothing renders:
1. Add a visible DOM overlay (not just console.log) to confirm mount executes
2. Log inside the scene's `update()` to verify it's being called
3. Trace from the render loop upward — if update() isn't called, the
   game loop itself may not be running

## Secondary Changes
- Wrapped `loadScene` body in try/catch for graceful error recovery
- Scene load failures now show a status message instead of leaving a
  broken state
- Minor comment cleanup in opening-cinematic (reduced redundant inline comments)
